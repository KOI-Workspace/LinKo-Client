# Design System

## Overview

조용하지만 분명한 인터페이스. Mintlify의 정돈된 라이트 UI 위에, 라벤더 보라와 핫 핑크라는 두 개의 시그널 컬러를 얹어 만든 디자인 시스템.

**한 줄로**: 흰 종이 위에 정확하게 떨어지는 보라색 잉크 한 방울.

핵심 원칙은 세 가지:
1. **흰 캔버스 우선** — 컬러는 메시지가 있을 때만 등장한다
2. **보라는 기능, 핑크는 감정** — 둘은 절대 동등하지 않다
3. **Pretendard 하나로 충분** — 폰트 다양성보다 위계의 명확함이 먼저다

## Colors

### 시그널 컬러 (Signal — 의미가 있는 색)

- **Primary `#8B5CF6`**: 활성 상태, 선택, 진행 중. "지금 여기" 신호
- **Primary Deep `#7C3AED`**: 강조 텍스트, 차트 막대, 호버 시 깊어지는 보라
- **Secondary `#EC4899`**: 액센트. 일러스트 디테일, 정보성 태그, 데코 라인. **CTA에는 절대 쓰지 않는다**
- **Error `#DC2626`**: 파괴적 액션, 검증 실패. 보라/핑크와 절대 섞지 않는다

### 베이스 컬러 (Base — 침묵하는 색)

- **Surface `#FFFFFF`**: 메인 캔버스. 비어있어 보이는 게 정상이다
- **Surface Soft `#FAFAFA`**: 사이드바, 보조 패널. 구역만 나누고 사라진다
- **On-surface `#0A0A0A`**: 본문. 검정에 가깝지만 완전한 검정은 아니다
- **On-surface Muted `#525252`**: 메타 정보, 설명문, 비활성 라벨
- **Border `#EAEAEA`**: 거의 보이지 않는 경계. 강조하지 말 것

### 컬러 사용 비율 가이드 (60-30-10 변형)

```
흰색·니어블랙 ████████████████████████████████  70%
보라 (Primary)  ████████░░░░░░░░░░░░░░░░░░░░░░  20%
회색 (Border·Muted) ████░░░░░░░░░░░░░░░░░░░░░░  8%
핑크 (Secondary) █░░░░░░░░░░░░░░░░░░░░░░░░░░░░  2%
```

핑크가 2%인 게 의도적이다. 한 화면에 핑크가 2개 이상 보이면 무언가 잘못된 것이다.

## Typography

**Pretendard 단일 패밀리**. 한글·영문·숫자가 한 호흡으로 흐른다.

### 위계

- **Display (32px / 700)**: 페이지 타이틀. "Good morning, hyunsoo" 같은 첫인사
- **Headline (20–24px / 600–700)**: 섹션 제목, 카드 헤더
- **Body (14–16px / 400)**: 본문 기본. 14px가 디폴트, 긴 글은 16px
- **Label (12px / 500)**: 카테고리 라벨, 메타 정보. 한글 라벨은 uppercase 변환하지 말 것 — 한글에는 대문자가 없다
- **Metric (32px / 600)**: 숫자 단독 표시. "Visitors: 6"의 6처럼

### Pretendard 미세 조정

- 한글은 자간 0, 영문 디스플레이는 `letter-spacing: -0.02em`로 살짝 좁힌다
- 250 이하 weight는 사용 금지 (흰 배경에서 가독성 무너짐)
- 본문 줄간격은 `1.5`가 표준, 한글 위주 단락은 `1.6`까지 허용

## Components

### Buttons

- **Primary**: 검정 필 + 흰 텍스트, `pill` 라운드. CTA의 무게감은 채도가 아닌 명도 대비에서 온다
- **Secondary**: 흰 배경 + 1px 보더, 동일 pill 라운드
- **Ghost**: 배경 없음, 호버 시에만 옅은 회색. 사이드바·아이콘 버튼용
- **Danger**: 빨강 필. 절대 보라와 같은 화면에서 메인 자리 차지하지 않게

> 버튼이 보라색이 아닌 이유: 보라를 CTA에 쓰면 활성 네비·차트·포커스 링과 충돌해서 "지금 어디를 눌러야 하는지"가 흐려진다. 검정이 시그널을 양보한다.

### Inputs

- 1px `#EAEAEA` 보더, 8px 라운드, 흰 배경
- 포커스 시 보라 보더 + 0.15 알파 보라 글로우 (3px)
- 플레이스홀더는 `#A3A3A3` — 본문보다 두 단계 옅게

### Cards & Panels

- 그림자는 `0 1px 3px rgba(10,10,10,0.06)`까지만. 떠오르는 게 아니라 **놓여있어야** 한다
- 12px 라운드, 1px 보더. 보더가 그림자 역할을 절반 이상 수행
- 카드 안의 카드는 만들지 말 것 — 중첩되면 즉시 답답해진다

### Badges

- 모두 pill 형태로 통일
- Success는 보라 톤(`#EDE9FE` 배경 + `#5B21B6` 텍스트). 그린은 사용하지 않는다
- Trialing/Info는 핑크 톤. 이게 핑크가 등장하는 거의 유일한 자리

## Do's and Don'ts

### Do
- ✅ **흰 공간을 두려워하지 말 것**. 여백이 럭셔리다
- ✅ **보라는 한 화면에 한 가지 역할**로 — 활성 네비면 활성 네비, 차트면 차트
- ✅ **호버 상태에 보라를 살짝 깊게** (`#7C3AED`) — 인터랙션의 보상감을 컬러 깊이로 표현
- ✅ **숫자는 크게, 라벨은 작게**. 정보 위계는 폰트 크기 비율로 만든다 (2.5배 이상 차이)
- ✅ **포커스 링은 항상 보라**. 브랜드는 디테일에서 드러난다

### Don't
- ❌ **핑크를 CTA에 쓰지 말 것**. 핑크는 말하는 게 아니라 속삭이는 색
- ❌ **보라 그라디언트로 도배하지 말 것**. 그라디언트는 일러스트·히어로 영역에 한정
- ❌ **검정 버튼과 보라 버튼을 같은 화면에 동등하게 두지 말 것**. 위계가 무너진다
- ❌ **그린·민트·청록을 추가하지 말 것**. 원본 Mintlify 흔적을 의도적으로 지웠다
- ❌ **Pretendard 외 다른 한글 폰트와 섞지 말 것**. 코드는 모노스페이스 허용, 그 외엔 금지
- ❌ **라운드와 직각을 한 컴포넌트 안에서 섞지 말 것** — pill이면 안쪽 요소도 둥글게

## Voice & Tone (시각적 톤)

이 디자인이 사람이라면:
- 말이 느리지 않지만 빠르지도 않다
- 농담은 잘 안 하지만 했을 때 핑크처럼 한 번에 꽂힌다
- 회의실에서 가장 먼저 말하지 않는다. 가장 정확하게 말한다

UI에서 이걸 어떻게 구현하나:
- 마이크로카피는 짧게. "Save changes" → "저장"으로 줄일 수 있으면 줄인다
- 아이콘은 outlined 스타일, stroke 1.5px. 두꺼운 아이콘은 너무 많이 말한다
- 애니메이션은 200–250ms ease-out이 디폴트. 느릿함은 신뢰감, 빠름은 가벼움

## Creative Latitude (창작 시 허용 범위)

이 시스템 위에서 새로운 화면을 만들 때, 다음은 **자유롭게 결정해도 된다**:

- 카드 내부 레이아웃 (그리드, 리스트, 매트릭스)
- 일러스트의 구체적 형태 — 단, 컬러는 보라 + 핑크 + 회색 3종만
- 마이크로 인터랙션 디테일 (호버 시 0.5px 살짝 들리기, 클릭 시 미세한 스케일다운 등)
- 데이터 시각화 그래프 종류 — 단, 시리즈 컬러는 `#7C3AED → #EC4899 → #525252` 순으로
- 빈 상태(empty state)의 메시지 톤 — 위트 허용, 단 핑크 톤으로 한 줄까지

다음은 **건드리지 말 것**:

- 컬러 토큰 추가 (그린·옐로우·블루 신규 도입 금지)
- 폰트 패밀리 추가
- 라운딩 스케일 변경 (sm/md/lg/pill 외 신규 값 금지)
- 그림자 농도 강화 (떠 있는 UI는 이 시스템과 맞지 않는다)

## When to use this MD vs JSON

- **JSON (`design.json`) 사용**: 컴포넌트 정확한 색상값/픽셀값/스펙을 적용할 때, 빌드 시스템에 토큰으로 주입할 때, 디자인 QA 체크리스트로 활용할 때
- **MD (`design.md`) 사용**: 새 화면 와이어프레임 그릴 때, 일러스트·마케팅 페이지 톤 결정할 때, 디자이너·개발자에게 "왜 이렇게 했는지" 공유할 때, AI에게 새 화면 만들어달라 요청할 때 컨텍스트로 첨부할 때
